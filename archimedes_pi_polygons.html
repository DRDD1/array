Explanation and history (Archimedes’ polygon method)

Archimedes’ approximation of π is one of the earliest examples of rigorous numerical estimation in mathematics. The key idea is geometric: for a circle, π is defined by the ratio of circumference to diameter. If we can trap the circumference between two quantities that we can compute, then we can trap π between two numbers as well.

Consider a circle of radius R. Its circumference is C = 2πR. Now place a regular polygon with n sides inside the circle (an inscribed polygon). Its perimeter P_in is strictly less than the circumference, because its straight edges cut inside the curved arc. This gives a lower bound:

Inscribed: P_in < 2πR ⇒ π > P_in / (2R)

Next, place a regular polygon with n sides around the circle so that each side is tangent to the circle (a circumscribed polygon). Its perimeter P_out is strictly greater than the circumference, because it wraps around the circle from the outside. This gives an upper bound:

Circumscribed: P_out > 2πR ⇒ π < P_out / (2R)

The strength of the method is that both perimeters can be computed from elementary geometry. For a regular n-gon with circumradius R, the central angle between adjacent vertices is 2π/n. The inscribed side is the chord length, so:

s_in = 2R sin(π/n), so P_in = n s_in = 2nR sin(π/n)

For the circumscribed polygon, each side is tangent to the circle, and the relevant triangle uses the tangent function:

s_out = 2R tan(π/n), so P_out = n s_out = 2nR tan(π/n)

Taking R = 1 (the choice used in this app), the bounds become especially simple: π lies between n sin(π/n) and n tan(π/n). As n increases, the polygons resemble the circle more closely and the gap between the bounds decreases. In modern terms, one can show the error shrinks on the order of 1/n² for large n, which explains why doubling n makes a noticeable improvement.

Historically, Archimedes did not write his method using sines and tangents (those came much later). Instead, he relied on careful geometric inequalities and repeated bisection of angles, effectively doubling the number of sides from a hexagon to a 12-gon, 24-gon, 48-gon, and finally a 96-gon. Using this, he produced the famous bounds 3 10/71 < π < 3 1/7, which correspond to approximately 3.1408 < π < 3.142857. That is a striking achievement given the tools available at the time.

From a computational perspective, there are two different “effort” stories. If you only want the numerical bounds and you allow modern functions like sin and tan, then computing the bounds is effectively constant time: evaluate sin(π/n), evaluate tan(π/n), then perform a few multiplications. In contrast, the visualisation in this page explicitly draws the polygon edges by plotting n vertices, so the rendering workload grows linearly with n. This is why extremely large values of n can be numerically cheap but visually expensive. Archimedes’ original approach was closer to an iterative doubling scheme, where each doubling step uses previously computed lengths and a small set of square root operations, which is closer to O(log n) steps if you only consider the sequence of doublings.

For more background, including the historical sources and later refinements, see the Wikipedia page: Approximations of π (Wikipedia).
