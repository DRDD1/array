Encrypted Text HTML Builder

Create a standalone encrypted HTML file from a plain .txt file or from text you paste here. The output file can be opened later in any modern browser, but it will still require the password every time it is reopened.

The encrypted output lets you read, copy, edit, and then lock and save the updated version again. No server is used. Encryption is done locally in your browser with Web Crypto. Best protection comes from using one long, strong passphrase.

1. Source text

Optional. You can also paste or type text directly.

This appears inside the encrypted output window.

2. Password and output

Use one strong passphrase, ideally 4 to 6 unrelated words or a high-entropy password from a password manager.

How it works

1. Choose a text file or paste text.

2. Set one strong passphrase.

3. Click Create encrypted HTML.

4. Save the generated HTML file.

5. Later, open that saved HTML file, enter the password, read or edit the text, then lock and save it again.

Security note: protection depends mainly on the strength of your passphrase. This file uses a slow password derivation step and AES-GCM, but weak passwords are still vulnerable. Suitable for personal offline use, not for very high-risk secrets.
