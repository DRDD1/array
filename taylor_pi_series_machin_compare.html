Explanation and history (Taylor series for arctan and π)

One of the cleanest ways to connect π to calculus is through the inverse tangent function. For |x| ≤ 1, the Taylor series of arctan(x) about x = 0 is an alternating power series:

arctan(x) = x − x^3/3 + x^5/5 − x^7/7 + … = Σ_{k=0}^{∞} (−1)^k x^{2k+1}/(2k+1).

Setting x = 1 gives arctan(1) = π/4, and therefore the classical Gregory–Leibniz series:

π/4 = 1 − 1/3 + 1/5 − 1/7 + … , so π ≈ 4 Σ_{k=0}^{N} (−1)^k /(2k+1).

The series is mathematically correct but numerically inefficient. The reason is visible in the term size: the (N+1)-th term has magnitude about 1/(2N+3), so the error decreases only proportionally to 1/N. This is a painfully slow route to high-accuracy π, but it is pedagogically valuable because it is easy to derive and it illustrates how convergence rates matter in computation. Because the series is alternating and its terms decrease in magnitude, it also has a simple rigorous remainder bound (a textbook result for alternating series):

If S_N = Σ_{k=0}^{N} (−1)^k /(2k+1), then |π − 4S_N| ≤ 4/(2N+3).

This bound explains why a small number of terms yields only a few correct digits. For example, if you want an error below 10^−6 using Gregory–Leibniz, you need roughly N on the order of two million terms. The point is not that the method is “bad”, but that it is not competitive when you care about efficiency. To improve convergence while staying within the same Taylor-series framework, mathematicians use identities that express π as a combination of arctangent values at smaller arguments. A widely used example is a Machin-style formula:

π = 16 arctan(1/5) − 4 arctan(1/239).

The key advantage is that when |x| is small, the series terms shrink like x^{2k+1}. For x = 1/5, each new term is smaller by roughly a factor of 1/25 compared with the previous odd power (up to the (2k+1) denominator). For x = 1/239, the shrinkage is enormous. The consequence is that the truncation error drops extremely quickly with N, even though Machin requires computing two arctan series and combining them. In practice, Machin-style formulas were historically important for high-precision hand computation of π before modern algorithms (such as Gauss–Legendre or the Chudnovsky formula) became standard for record-setting digits.

This app compares both approaches at the same truncation order N. That is a fair comparison of convergence: each method gets the same number of Taylor terms. You will typically observe that Machin reaches many more correct digits for modest N. The computational effort panels show a crude arithmetic count: Machin roughly doubles the per-term work (two series), but can reduce the required N by orders of magnitude for a target accuracy. This is an important lesson in numerical analysis: reducing N is often the dominant win, even if each step is a little more expensive.
