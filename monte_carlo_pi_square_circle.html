Description and background (Monte Carlo probability method)

The Monte Carlo estimate of π is a classic demonstration of how geometry and probability can be combined to estimate a constant. It is not competitive with modern high-precision algorithms for π, but it is conceptually powerful because it relies on very little mathematics. The method begins with a simple geometric fact: if you draw a circle of radius 1 inside a square of side 2, then the circle’s area is π and the square’s area is 4. This leads immediately to a probability statement. If you pick a point uniformly at random in the square, the probability that it lands inside the circle is equal to the ratio of areas:

p = P[(x,y) is inside circle] = Area(circle) / Area(square) = π / 4.

Therefore, if you generate N independent random points and count how many fall inside the circle, say M, then M/N is an empirical estimate of p. Rearranging gives an estimate of π:

π ≈ 4 (M / N).

The core computation is a membership test: a point (x,y) is inside the circle if x²+y² ≤ 1. Everything else is bookkeeping. This makes the method attractive for teaching, because it is easy to implement, easy to visualise, and easy to reason about. You can literally watch the estimate settle as the cloud of points fills the square. The method also highlights the idea that randomness can be used as a numerical tool, which is the defining theme of Monte Carlo methods across scientific computing, from radiative transport to Bayesian inference.

The weakness of the approach is its convergence rate. The estimate is essentially based on a binomial proportion. If each point is an independent trial with success probability p≈π/4, then M has variance N p(1−p), so the standard deviation of the fraction M/N is √(p(1−p)/N). Multiplying by 4 to obtain π gives a characteristic statistical uncertainty

σ_π ≈ 4 √(p(1−p)/N).

This implies that error typically decreases like 1/√N. That is slow. To gain one extra decimal digit of accuracy you need roughly 100 times more samples. This is very different from methods such as Machin-like formulas or modern rapidly convergent algorithms, where each additional term can buy multiple digits. Nevertheless, Monte Carlo remains valuable because it extends to high-dimensional integrals where “clever” deterministic quadrature becomes impractical. In those settings, the slow 1/√N convergence can still be the best available option, and variance-reduction techniques are used to improve efficiency.

In this app you can choose N and watch the estimate evolve. Because drawing every point can become visually dense, the display can be “thinned”: the app may skip drawing some points while still counting them in the estimate. If you supply a seed, the pseudo-random generator becomes deterministic, which is useful for repeating runs and comparing settings. For a more detailed overview of Monte Carlo methods and the π example, see: Monte Carlo method (Wikipedia) and Approximations of π: Monte Carlo methods (Wikipedia).
